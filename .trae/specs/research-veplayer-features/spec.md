# 调研火山视频播放器核心能力 Spec

## Why
需要深入了解火山引擎 Web 播放器 SDK（VePlayer）的核心能力，特别是自定义 UI 交互方面的支持程度。同时，需要梳理现有 `/apps/replay` 项目中关于多分辨率播放、小窗视频播放以及主副视频同步的实现机制，以便为后续基于 VePlayer 重构或增强现有播放器功能提供理论依据和技术选型参考。

## What Changes
- 调研并总结火山视频播放器的核心基础功能与进阶功能。
- 重点梳理火山视频播放器的自定义 UI 与交互能力（如自定义样式、图标替换、CSS 覆盖、插件机制等）。
- 深入分析 `/apps/replay` 项目中的 `PlayerModal.tsx` 组件，梳理其实现多分辨率切换、画中画（小窗）播放、以及主副视频绝对时间同步的具体代码逻辑。
- 输出一份完整的调研报告总结。

## Impact
- Affected specs: 无直接代码影响，仅输出调研结果，可能影响后续播放器重构相关的 spec。
- Affected code: 无。

## ADDED Requirements
### Requirement: 播放器能力调研报告
系统 SHALL 提供一份详尽的调研报告，涵盖火山播放器的 UI 自定义方案、内置插件生态，以及现有业务系统中的视频同步与多清晰度切换原理。

#### Scenario: Success case
- **WHEN** 开发者需要评估使用 VePlayer 替换或升级现有播放器功能时
- **THEN** 可以参考该调研报告，快速了解 VePlayer 的定制能力以及如何与现有绝对时间同步逻辑进行融合。
